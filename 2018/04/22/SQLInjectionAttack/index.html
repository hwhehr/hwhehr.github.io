SQL注入攻击

  • 宽字节注入

  宽字节注入原理：

  GBK 占用两字节，ASCII占用一字节。PHP中编码为GBK，函数执行添加的是ASCII编码，MYSQL默认字符集是GBK等宽字节字符集。

  在客户端输入字符'被PHP转义成\'，\'编码是%5c%27。

  如果注入的时候在'前面加上%df字符变成%df'，经过转义之后变成%df%5c%27，而之前说了MYSQL默认的是GBK等宽字节字符集，GBK编码格式会把本来属于ascii编码的字符‘吃掉’，会将%df%5c编码成運，所以会将%df%5c%27编码成運'。

  于是就可以继续愉快的注入了。

  注：

    1.宽字节注入与HTML页面编码是无关的，在<meta charset=utf8>情况下一样有可能进行宽字节注入。

    2.宽字节注入不一定只有GBK编码，宽字节编码都有可能可以进行宽字节注入。

  例：

  BugkuCTF–sql注入:http://103.238.227.13:10083/

  使用'没有反应，于是猜测是宽字节注入。

  这道题里面还有一个知识点（keng）就是key是MYSQL的关键字，直接使用会报错。MYSQL提供了一个转义符号 ` 来区别关键词和库表字段名。

  • UNION语句联合查询

  要使UNION操作符正确工作，需满足下列要求：

   ◦ 两个查询返回的列数必须相同；

   ◦ 两个SELECT语句对应列返回的数据类型必须相同（或至少是兼容的）。

  所以用UNOION构造SQL语句的时候需要进行两部猜解：匹配列数与匹配数据类型。

1. 匹配列

  一种方法是一位一位的尝试；

  还有一种方法是使用使用order by子句查询，当列数很多的时候，这种方法会在数据库中报错更少，留下痕迹更少。

2.匹配数据类型

  • 枚举数据库

  在SQL Server、MySQL以及Oracle等数据库中，都有会有一个库存有所有库的库名，以及其中的表名和列名。可以通过这项特性查询到敏感的或者我们感兴趣的一些数据库。

  以MySQL为例：

  information_schema不仅包含了数据库结构，还包含与数据库用户权限及其得到的授权信息。例如：

SELECT grantee, privilege_type, is_grantable FROM information_schema.user_privileges;

  若想知道不同数据库授权用户权限，可以使用下列查询：

SELECT grantee, table_schema, privilege_type FROM information_schema.schema_privileges;

  • INSERT查询注入攻击

  1.插入用户规定数据

  假定服务端构造的SQL语句是这样的：

INSERT INTO table (firstname, lastname) VALUES ('$firstname','$lastname')

  可以构造这样的语句：

  这条语句执行了下列几个操作：

  1.插入任意值并且使用单引号关闭该字符串；

  2.为了在第二列插入数据，语句注入了一个子查询，该查询将第一个数据库用户的用户名和密码的哈希连接成一个字符串。其中还使用fn_varbintohexstr()函数将二进制的哈希值转换为十六进制格式。

  3.关闭了所有需要关闭的圆括号，并注释掉其余的查询。

  但是当可注入的参数是最后一个参数时，就很麻烦了，因为无法关闭前面一个参数，去构造下一个参数。在MySQL处于ANSI模式（或者任何实现了PIPES_AS_QUOTES的其他模式）时，可以用||符号拼接字符：

INSERT INTO table (firstname, lastname) VALUES ('firstname','lastname' || (select @@version)) --

  但是在PIPES_AS_QUOTES时（比如TRADITIONAL模式），那么||操作符会被当成OR逻辑符号。

  于是我们可以运用MySQL的一个特性，当一个整数与一个字符相加时，整数拥有更高的优先级：

  接下来就可以使用ascii，substring等函数进行注入。

  • 堆叠注入

  堆叠查询是指在单个数据库连接中执行多条查询语句，堆叠注入是终止原始查询，再添加一条全新的查询语句，促使远程服务器执行多条语句；

1'; exec master..xp_cmdshell 'DEL important_file.txt' ### SQL Server

  但并非所有数据库服务平台都支持堆叠查询，根据远程数据库服务器的差别和所使用的技术框架不同，情况各有不同。例如，使用 ASP.NET 和 PHP 访问 Microsoft SQL Server 时允许堆叠查询，但是如果使用Java来访问，就不允许。使用PHP访问PostgreSQL时，PHP允许堆叠查询；但是如果访问MySQL，PHP不允许堆叠查询。

  在Oracle数据库中有一个特殊的堆叠注入的漏洞，Oracle SQL语句是不允许执行堆叠查询的。所以需要执行PL/SQL块，PL/SQL程序设计语言是直接内置于Oracle中，可以使用一个匿名PL/SQL块，它包含在一条BEGIN语句与一条END语句之间。

  用户基本都有权限访问这两个函数，可以使用这两个函数执行DML/DDL语句块，例：

  1. 使用大小写变种

  2. 使用SQL注释

'/**/UNION/**/SELECT/**/password/**/FROM/**/table/**/WHERE/**/username/**/LIKE/**/'admin'--

  3. 使用URL编码

  4. 使用动态查询执行

  这种方法主要运用在Oracle、SQL Server等数据库中，具体不清楚怎么使用，先码在这。

  当SELECT被过滤的时候可以使用 'SEL'||'ECT' CHAR(83) +CHAR(69) +CHAR(76) +CHAR(69) +CHAR(67) +CHAR(84)

  MySQL中可以用 WHERE data = 0x64617461 代替 WHERE data = 'data'

  5. 使用空字节

  有的网站会有入侵检测系统（IDS）或WAF这些检测机制，而这些组件通常是由原声语言写的（如C++），这些语言的字符串通常是由0x00作为停止符。所以在提交SQL注入语句的时候可以加上%00来“结束”字符串。

%00' UNION SELECT password FROM table WHERE username='admin'--

  6. 嵌套剥离后的表达式

  有些审查过滤器会先从用户输入中剥离特定的字符串，然后在处理数据，如果没有递归多次剥离的话，可以尝试这种方法：

SELSELECTECT

  7. 利用截断

  审查过滤器通常会对用户提供的数据执行多种操作，有时会将输入的数据截断成最大长度，比如：

    （1）将引号双重编码，使用"代替';

    （2）将每一项截断成16个字符；

SELECT uid FROM user_table WHERE username = 'name' AND password = 'pw'

  提交 username : aaaaaaaaaaaaaaa' , password : or 1=1--

  username有15个a，最后一个'会双重编码成两位字符，username变成一个包含了17个字符的字符串，接下来将其截断成16个字符，清除附加的单引号。这样可以通过向查询插入一个保留的引号字符来干预其语法。
